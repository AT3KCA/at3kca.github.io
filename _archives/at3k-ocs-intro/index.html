AT3K-OCS

正如主页所说，AT3K-OCS 是我为了方便管理其所有md文档而进行的前端尝试，在此之前，我从未涉猎过UI设计或文档整理。从你看到的AT3K-OCS实现效果来说，还算不错。由此可以得出，创建个人页面是简单（当然你需要付出一点时间）且个性化的。

个人页面，重点在个人而不是页面，如果使用千篇一律的模板会让你丧失个性，互联网中，最不缺少的就是副本。我当然承认一个好看的模板会让没有自己页面的人们惊叹，不过，为了留下自己，我们还是自己尝试一下吧。我也是第一次创建页面，也许会更接近第一次尝试。

是的，只有自己才知道自己需要什么，明确了自己的需求，才能做出适合自己的工具。

用到的东西

  • Linux (WSL2)

    • vim
    • nodejs

  • Nextjs

开始一切

很幸运的，NextJS为我们提供了一个快速创建项目的通道，在这之前，你需要安装nodejs，这里以Ubuntu为例

apt install nodejs

如果apt有类似于无权限的提示，试着在前面加上 sudo 。

然后我们可以使用nextjs项目创建向导来帮助我们开启一切。

停下

这里应该是介绍AT3K-OCS的开发理念而不是直接贴上代码，这种事应该在别的地方做

所以

AT3K-OCS是以markdown文档显示为基础的，通过加入其它http后端并专门添加页面进行fetch的一种类似服务提供者的页面，我可以在这里添加写好的后端，发布markdown文档之类的活动。甚至说，如果我想，我甚至可以写一个展示Galgame的后端。AT3K-OCS这一模式很让我激动，你可以把一切你想干的事放在一个网站进行。

相似的理念，统一的样式，顺手的操作，完成任务都令人如此开心，AT3K-OCS是一个可以复制的蓝图。
